设计说明

方案 A · 横向卡片 + 大圆环

  • 大圆环（88px）：SVG stroke-dasharray 实现，内嵌 48px 头像圆；圆环展示该客户所有合同的总体付款进度
  • 圆环下方：总进度百分比 + 合同总金额 — 一眼抓住关键数据
  • 右侧信息密度高：客户名 + 合同数 pill → 联系方式 → 合同列表（每行含小圆环进度 + 业务标签 + 金额）
  • 信息密度提升：横向布局减少了纵向空间浪费，190px 高度承载了原 360px 的全部信息 + 更多数据
  • 适合宽屏：min-width 520px，桌面端 2~3 列排列

方案 B · 竖向卡片 + 小圆环

  • 44px 大头像：无圆环包裹，更简洁
  • 行内小圆环（30px）：替代线性进度条，每份合同的进度用小圆环展示，视觉上更紧凑
  • 底部汇总条：显示合同数 + 总金额 + 总进度百分比
  • 适合窄屏/移动端：min-width 320px，可 3~4 列排列

圆环进度计算公式

圆周长 C = 2πr = 2 × 3.14159 × 42 ≈ 263.9
stroke-dasharray = "263.9"
stroke-dashoffset = 263.9 × (1 - progress%)
示例：65% 进度 → offset = 263.9 × (1 - 0.65) = 92.4

颜色语义

绿色 = 已完成 (100%) 金色 = 部分付款 (1-99%) 灰色 = 未付款 (0%) 红色 = 总进度低 (<40%) 大圆环告警
