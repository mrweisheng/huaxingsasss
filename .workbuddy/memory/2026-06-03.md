  - 无合同时显示"暂无关联合同"占位
  - 加载中显示骨架屏
  - 联系人信息改为紧凑一行显示
- **CustomerList.css**: 新增合同行、进度条、业务类型徽章、状态圆点、骨架屏等样式；卡片最小宽度调整为360px

## DeepSeek → 百炼 DashScope 迁移

将所有 DeepSeek API 调用从硅基流动（SiliconFlow）迁移到阿里云百炼 DashScope 平台。

### 变更范围（6个文件）
1. **`.env` / `.env.example`**: 移除 `DEEPSEEK_BASE_URL`、`DEEPSEEK_MODEL`、`DEEPSEEK_AGENT_MODEL`；新增 `DASHSCOPE_AGENT_MODEL=deepseek-v4-flash`（复用 `DASHSCOPE_API_KEY`）
2. **`config.py`**: `DEEPSEEK_*` 配置项改为 `DASHSCOPE_AGENT_MODEL` + `AGENT_MAX_RETRIES` / `AGENT_RETRY_BASE_DELAY`
3. **`llm_client.py`**: `DeepSeekClient` → `DashScopeAgentClient`，API key 改用 `DASHSCOPE_API_KEY`，通过兼容模式 API 调用（`DASHSCOPE_BASE_URL/chat/completions`），保持流式 SSE + 工具调用 + 指数退避重试
4. **`agent.py`**: 导入 `DashScopeAgentClient`，日志/注释/`llm_model` 字段全部更新
5. **`tools.py`**: PDF 文本解析改用 `dashscope.Generation.call()` 原生 SDK（`enable_thinking=True`）
6. **`contract_analyzer.py`**: 修正误导注释（实为 DashScope qwen-plus，非 DeepSeek）
7. **`CLAUDE.md` / `SPEC.md`**: 文档更新

### 关键设计决策
- Agent 流式推理部分使用兼容模式 HTTP API（需工具调用 + 流式 SSE，兼容模式最稳定）
- PDF 文本解析部分使用原生 `dashscope.Generation` SDK（同步调用，支持 `enable_thinking`）
- `DASHSCOPE_API_KEY` 与 qwen3-vl-flash 共用，不需要额外 API key
