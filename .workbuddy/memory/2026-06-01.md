## Agent 聊天助手 - 图片上传后无回复 Bug 修复

### 问题现象
用户在前端聊天助手上传图片后，LLM 调用 `analyze_image` 工具成功返回结果，但前端界面停留在"好的，我来分析这张图片"不再继续。

### 根因分析
**agent.py ReAct 循环中，第二轮 LLM 调用（工具结果返回后）可能返回空文本，原因：**
1. 用户消息为自动生成的"请分析上传的图片内容，提取关键信息并总结"，但系统 prompt 要求凭证走 `match_receipt` 流程
2. LLM 在"总结"和"继续调 match_receipt"之间犹豫，最终返回空文本
3. `full_text=""`、`tool_calls=[]` → 代码分支到"无工具调用"→ 保存空内容并 yield done
4. 前端收到 done 事件但无新文本可显示 → 界面"没有下文"

### 修改内容（agent.py）
1. **每轮 ReAct 迭代增加 logging**：`ReAct迭代 #N: messages=X 条` 和 `ReAct迭代 #N 完成: full_text_len=X, tool_calls=X`
2. **LLM 调用增加 try/except**：捕获异常后尝试用工具结果生成兜底回复
3. **新增 `_build_tool_result_fallback`**：从最近的 tool 消息中提取结构化数据（凭证/合同/文档），组装为可读的自然语言回复
4. **空文本检测**：第二轮及以上若 LLM 返回空文本且无工具调用，自动触发兜底回复
