## Agent 聊天助手 - 图片上传后无回复 Bug 修复

### 问题现象
用户在前端聊天助手上传图片后，LLM 调用 `analyze_image` 工具成功返回结果，但前端界面停留在"好的，我来分析这张图片"不再继续。

### 根因分析
**agent.py ReAct 循环中，第二轮 LLM 调用（工具结果返回后）可能返回空文本，原因：**
1. 用户消息为自动生成的"请分析上传的图片内容，提取关键信息并总结"，但系统 prompt 要求凭证走 `match_receipt` 流程
2. LLM 在"总结"和"继续调 match_receipt"之间犹豫，最终返回空文本
3. `full_text=""`、`tool_calls=[]` → 代码分支到"无工具调用"→ 保存空内容并 yield done
4. 前端收到 done 事件但无新文本可显示 → 界面"没有下文"

### 修改内容（agent.py）
1. **每轮 ReAct 迭代增加 logging**：`ReAct迭代 #N: messages=X 条` 和 `ReAct迭代 #N 完成: full_text_len=X, tool_calls=X`
2. **LLM 调用增加 try/except**：捕获异常后尝试用工具结果生成兜底回复
3. **新增 `_build_tool_result_fallback`**：从最近的 tool 消息中提取结构化数据（凭证/合同/文档），组装为可读的自然语言回复
4. **空文本检测**：第二轮及以上若 LLM 返回空文本且无工具调用，自动触发兜底回复

### 涉及文件
- `D:\CODE PROJECT\huaxingsasss\backend\app\ai\agent.py` — ReAct 循环 + 新增 `_build_tool_result_fallback` 方法

## 凭证路径双保险 - 代码审核后的加固修复

### 问题 1：DB 兜底路径可被重复消费
`_get_receipt_image_path` 第 3 层（DB 查询）没有消费标记，同一会话里如果多次创建付款，会重复关联同一张图片。修复：在 `ChatHistory.extra_metadata` 中加 `receipt_consumed: true` 标记，查询时跳过已消费记录。

### 问题 2：临时文件不存在时写入无效路径
`_ensure_file_in_receipt_dir` 在文件不存在或复制失败时原本返回原路径（如 `agent_upload/xxx`），这个路径实际不存在，会导致付款记录关联了一个虚假路径。修复：改为返回 `None`。`update_payment` 中检测到 `None` 时删除 `receipt_image_path` key，避免用空值覆盖已有路径。

### 修改内容（tools.py）
1. `_get_receipt_image_path`：DB 兜底返回路径前，检查并写入 `receipt_consumed` 标记
2. `_ensure_file_in_receipt_dir`：文件不存在和复制异常时返回 `None` 而非原路径
3. `update_payment`：解析结果为 `None` 时移除 `receipt_image_path` key
4. 类型注解 `_ensure_file_in_receipt_dir` 返回值改为 `Optional[str]`

## 以合同货币为基准：同币种跳过 CNY 换算

### 问题
所有付款强制换算 CNY，同币种付款也显示无意义的折算金额。合同 HKD 付 HKD 50000 显示 ¥43,560。

### 设计原则
- 合同货币为基准：合同用什么币种，系统就以什么币种跟踪
- 同币种不折算：付款币种 == 合同币种时，`amount_in_cny`/`exchange_rate` 设为 None
- 混币种自动结算：付款币种 ≠ 合同币种时，按付款日实时汇率换算，自动累计合同金额
- 合同完成状态：统一用原币 `paid_amount >= total_amount` 判断

### 修改内容

**`payment_service.py`** — 核心逻辑：
1. `create_payment_with_exchange_rate`：新增 `currency == contract.currency` 判断，同币种跳过换算
2. `_add_to_contract_paid`：CNY 字段改为有条件更新，完成判断统一用原币
3. `_add_to_contract_expense`：CNY 字段改为有条件更新
4. `update_payment`：保持 None 语义传递给累加方法

**`tools.py`** — 工具定义：
1. `create_payment`：删除 currency 的 `"default": "CNY"`，更新描述
2. `create_expense`：删除 currency 的 `"default": "CNY"`
3. `create_contract`：删除 currency 的 `"default": "CNY"`，更新描述

**`prompts.py`** — 提示词：
1. 原"汇率自动折算为人民币"改为"同币种不折算，混币种才结算"
2. 新增"币种确认规则"：不清楚必须问用户，不可猜测
3. analyze_image receipt/contract prompt 更新 currency 提取说明

**前端**：
1. `PaymentList.tsx`：`amount_in_cny` null 时显示 `-`
2. `ContractDetail.tsx`：同上
