# 2026-06-02 工作日志

## Bug Fix: Agent 路径 create_contract 丢失 signed_date，以及根治重构

**问题**：通过 AI Agent 对话创建合同时，签订日期（signed_date）偶发性丢失。买车合同 HT202606021145559EC1 的 signed_date 为空，但两地牌合同 HT20260602114813E545 有值。

**根因**：`backend/app/ai/tools.py` `create_contract` 方法中，VL→Agent→代码 的三层数据流中，各字段的回退策略不一致——有些从 merged（VL缓存）回退，有些只读 kwargs（Agent传参）。`signed_date`/`start_date`/`end_date` 属于后者，Agent 没传就丢了。

**修复 + 重构**（tools.py）：
1. 引入 `_resolve(kwargs_key, merged_path, parser, default)` 统一取值函数：Agent 显式传入优先，否则走 VL 缓存回退路径
2. 用声明式 `contract_fields` 字典替代零散的字段赋值，新增 VL 字段只需加一行
3. `start_date`/`end_date` 额外处理 `validity_period` 嵌套路径（VL 输出结构如此）
4. `_parse_date()` 兼容 date 对象、字符串、datetime 多种格式

**根治原则**：VL 已正确提取的数据，不应因 Agent 漏传参数而丢失。所有 VL→ContractField 映射都应有缓存回退。

**仍然存在的已知 Gap**：
- `confidence` 模型字段：VL 有提取、存入了 contract_data JSON，但从未写入 `contract.confidence` 列（两条路径都如此）
- `needs_review` 模型字段：同样未被设置

## 第二轮修复：confidence + needs_review 补齐

**涉及文件**（3 个）：
1. `backend/app/ai/tools.py` — Agent 路径 create_contract：从 merged 提取 confidence，设置 contract.confidence 和 contract.needs_review（<0.85），同时加入返回值
2. `backend/app/tasks/contract_tasks.py` — REST API 路径：设置 contract.confidence 和 contract.needs_review
3. `backend/app/services/contract_service.py` — update_contract_data 路径：同样补齐

**确认**：API 层 `contracts.py:175` 已在读取 `contract.confidence` 用于展示，修复有价值。

**审计结论**：VL 输出 20 个字段 → Contract 模型 15 个列，映射已全部闭环，无遗漏。

## 第三轮：审核反馈修复（外部审核意见）

**问题 1（设计冲突）**：Agent 路径 vs Celery 路径置信度→状态规则不一致。
→ **不接受修复**。两条路径语义不同：Agent 路径有人工确认（用户看过分析结果并同意录入），应直接 active + needs_review 标记风险；Celery 路径无人把关，pending_review 是合理的。当前设计正确。

**问题 2（真实 bug）**：total_amount 兜底 0 会"静默成功"。
→ **已修复**。contract.total_amount == 0 时返回 JSON 增加 `_warning` 字段，提示 Agent 告知用户核实金额。

**问题 3（设计缺陷）**：_resolve 返回 default 时不应用 parser。
→ **已修复**。default 也走 parser 保证类型一致。
