## 修复登录页输入框白色外观与深色背景不协调问题

**问题根因**：`index.css` 中全局 `.ant-input-affix-wrapper` 样式（白色背景/亮边框）覆盖了登录页 `.l2-glass-input` 的玻璃态输入框样式，特异性不足。

**修复方案**：
- 使用双类名 `.l2-glass-input.l2-glass-input .ant-input-affix-wrapper` 提升选择器特异性
- 覆盖 background、border-color、box-shadow、caret-color 等全部相关属性
- 同步覆盖 prefix icon、placeholder、密码眼睛图标、focus 状态
- 为表单错误提示文字（`.l2-login-form .ant-form-item-explain-error`）添加柔和红色（`rgba(252,165,165,0.85)`），避免纯红在暗色背景刺眼
- 登录 Form 添加 `className="l2-login-form"` 

**文件**：
- `frontend/src/index.css`（`.l2-glass-input` 样式块重写 + `.l2-login-form` 新增）
- `frontend/src/pages/Login.tsx`（Form 添加 className）

## 重构合同卡片金额区域UI

**问题**：旧 amount-section 是灰色背景框，总金额/已付/尾款层级混乱，进度条和金额不直观，笔数信息在金额区外割裂

**设计思路**：
- **总金额** → 顶部渐变背景行，作为视觉锚点，直接用大字号+粗体突出
- **已付/未付** → 双栏并排，用颜色圆点+标签清晰标识，数值直接对比
- **进度集成** → 进度条从"各自独立"改为仅放在"未付"栏内，搭配百分比，节约空间
- **笔数信息** → 集成到"已付"栏内，格式为 `X/Y笔`，支出笔数以红色 badge 展示
- **移除** 旧的 payment-summary 独立行

**采用新的 class 名**（全部重构，不保留旧名）：
- `.amount-hero` / `.amount-hero-label` / `.amount-hero-value` — 总金额
- `.amount-split` / `.amount-split-item` / `.amount-split-vert` — 双栏容器
- `.split-item-header` / `.split-dot` / `.split-label` — 栏头
- `.split-value` / `.split-meta` / `.split-count` / `.split-expense` — 数值+元信息
- `.split-progress` / `.split-progress-track` / `.split-progress-fill` / `.split-progress-text` — 进度条

**文件**：
