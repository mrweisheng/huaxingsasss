# 无凭证支出改造方案 — 评估与实施计划

## 评估结论：方案可行，3 处需调整

原方案整体优秀，CLAUDE.md 铁律对齐准确，改动范围克制。以下是逐项评估和调整。

---

### ✅ 方案优势

1. **铁律对齐正确**：LLM 做判断（场景识别/字段提取/追问），代码做边界（三道栅栏）
2. **改动克制**：不动收入路径、不动 DB schema、不动 Service 层，复用现有 `notes` 做审计标记
3. **回退成本低**：删除 `no_receipt` 参数 + 前端 chip 即可完整回滚

### ⚠️ 需调整的 3 处

#### 调整 1：`create_payment_record` 中 no_receipt 分支的落库位置

**原方案**：在 `has_receipt = bool(receipt_data or receipt_image_path)` 的 `if has_receipt` 块后追加 `elif no_receipt`

**问题**：当前代码结构是 `if has_receipt: ...`，后面没有 `else`。无凭证支出需要走 `PaymentService.create_payment_with_exchange_rate`，该方法内部根据 `has_receipt`（即 `receipt_image_path`）决定 status=pending 还是 paid。no_receipt 场景下 `receipt_image_path` 为 None，service 层会创建 pending 记录，所以需要在创建后手动更新 status。

**调整方案**：保持原方案逻辑，但落库代码应放在 `has_receipt` 块之后、`self.db.commit()` 之前。具体位置在 `tools_v2.py:816`（`has_receipt = bool(receipt_data or receipt_image_path)`）后追加 `elif no_receipt:` 分支。

#### 调整 2：ContractDetail.tsx 的 chip 插入位置

**原方案**：在 `cd-pay-card-desc` 区域插入

**调整**：更合适的位置是 `cd-pay-card-meta` 区域（line 266-303），在凭证按钮之后、description 之前。这里已经有 `receipt_image_path` 的判断逻辑，插入 no-receipt chip 逻辑上更连贯。

#### 调整 3：PaymentList.tsx 的 chip 插入位置

**原方案**：在金额列追加 chip

**调整**：金额列已有 CNY 折算 div，再加 chip 会拥挤。建议在"业务说明"列（`description`，line 284）的 render 中，如果 notes 以 `[无凭证支出]` 开头，在 description 文字旁显示 chip。

**改动点 B** — 三道安全栅栏（在 `has_receipt = bool(...)` 之前，约 line 815）：
```python
# ── 无凭证支出：三道栅栏 ──
if no_receipt:
    if type != "expense":
        return json.dumps({"error": "no_receipt 仅支持支出（type=expense）"}, ensure_ascii=False)
    missing = []
    if not payee_name:
        missing.append("收款方 payee_name")
    if not description:
        missing.append("用途说明 description")
    if not paid_date:
        missing.append("付款日期 paid_date")
    if missing:
        return json.dumps({"error": f"无凭证支出必须明确：{', '.join(missing)}。请先向用户追问。"}, ensure_ascii=False)
    if receipt_image_path or receipt_data or receipt_file_ids:
        return json.dumps({"error": "no_receipt=true 时不得同时传凭证字段"}, ensure_ascii=False)
```

**改动点 C** — 落库分支（在 `if has_receipt:` 块之后，约 line 837）：
```python
elif no_receipt:
    needs_contract_update = payment.status == "pending"
    payment.status = "paid"
    base_note = notes or description or ""
    payment.notes = f"[无凭证支出] {base_note}".strip()
    if needs_contract_update:
        amt_cny = payment.paid_amount_in_cny or payment.paid_amount
        if type == "expense":
            PaymentService._add_to_contract_expense(
                self.db, contract, payment.paid_amount, payment.currency,
                amt_cny, payment.paid_date,
            )
        else:
            PaymentService._add_to_contract_paid(
                self.db, contract, payment.paid_amount, payment.currency,
                amt_cny, payment.paid_date,
            )
```

**改动点 D** — `TOOL_DEFINITIONS` 中 `create_payment_record` 的 parameters（约 line 1186）：
- 在 properties 中追加 `no_receipt` 字段定义

### 文件 2：`backend/app/ai/prompts_v2.py`

在 `build_system_prompt` 中"确认规则"块之后、"业务背景"之前，追加"无凭证支出场景识别"段落：

```python
### 无凭证支出场景识别

当用户在录支出会话中发送纯文字描述（无图片附件、未提到上传凭证），按以下流程：

1. **必收字段提取**：
   - 金额（amount）
   - 币种（currency，HKD/CNY，必须明确，不允许默认）
   - 收款方姓名（payee_name）
   - 用途简述（description，10-20字，如"介绍客户返点"、"停车费"）
   - 付款日期（paid_date）—— **特别强调**：用户没说就必须追问"是哪一天给的？"，绝对不要假设"今天"

2. **任一字段缺失** → 先列出已知字段，逐一追问缺失项，得到答复再继续

3. **字段齐全** → 列出确认计划，格式示例：
   即将录入【无凭证支出】，请确认：
   - 合同：[合同名]
   - 收款方：张三
   - 金额：HK$500
   - 日期：2026-06-12
   - 用途：介绍客户返点
   - 来源：用户口头确认（无凭证）
   是否确认录入？

4. **用户确认后**调用 `create_payment_record(type="expense", no_receipt=true, ...)`
```
