# 统一前端压缩，移除后端压缩

## 目标
- 前端统一压缩：最长边 1600px, quality=85, JPEG
- 后端移除压缩逻辑，专注业务
- 所有上传入口都走压缩

## 改动清单

### 1. `frontend/src/utils/imageCompress.ts`
- `maxEdge`: 1920 → 1600
- `quality`: 0.8 → 0.85

### 2. `frontend/src/components/ReceiptChatModal.tsx`
- 两个 Upload 的 `beforeUpload` 里加 `compressImage(file)` 调用（与 AgentChat.tsx 对齐）

### 3. `frontend/src/components/ContractChatModal.tsx`
- Upload 的 `beforeUpload` 里加 `compressImage(file)` 调用

### 4. `backend/app/services/file_analyzer.py`
- `_classify_file`: 移除 `compress_image` 调用，直接把 `file_bytes, mime` 传给 `call_vl_model`
- `_analyze_content`: 图片分支移除 `compress_image` 调用，直接传 `file_bytes, mime`
- `_analyze_content`: 扫描件 PDF 分支移除 `compress_image` 调用
- 移除 `compress_image` 的 import（如果不再使用）

### 5. `backend/app/utils/file_analysis.py`
- `compress_image` 函数保留（不删），但不再被调用
- 或者：如果确认无其他调用方，可以删除

## 不改的地方
- `call_vl_model` 本身不涉及压缩（它只接收 bytes+mime）
- `llm_client.py:VisionModelClient.parse_contract_image` 本来就没压缩，不用动
- `config.py` 默认值改不改都行（`.env` 会覆盖）

## 验证
- 前端 `npm run build` 确认无报错
- 后端 `PYTHONIOENCODING=utf-8 uv run pytest` 确认无报错
- 手动测试：AgentChat / ReceiptChatModal / ContractChatModal 上传图片，确认 VL 分析正常返回
