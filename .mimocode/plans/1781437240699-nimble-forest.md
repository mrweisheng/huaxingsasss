# 凭证录入 description 自动填充修复计划

## 背景

### 现象
用户录入收入时上传收据凭证，系统自动识别后生成的 `description` 字段值为"两地牌相关费用"，但收据上明确写着"该款项系付：深圳湾现牌定金"。

### 期望
`description` 应该提取收据上"该款项系付"字段的具体内容，如"深圳湾现牌定金"，而不是笼统的业务类型推断"两地牌相关费用"。

---

## 原因分析

### 问题根源
`RECEIPT_ANALYSIS_PROMPT`（`backend/app/ai/prompts.py:259-280`）中缺少收据"该款项系付"字段的提取指令。

### 当前逻辑链路

```
收据图片 → analyze_files → FileAnalyzer → RECEIPT_ANALYSIS_PROMPT → LLM 返回 JSON
                                                                    ↓
                                                              receipt_data
                                                                    ↓
                                          match_and_confirm_payment → _build_payment_description()
                                                                            ↓
                                                                       description
```

### 关键代码位置

**1. 收据分析提示词** (`backend/app/ai/prompts.py:259-280`)
```python
RECEIPT_ANALYSIS_PROMPT = """...{
  "business_hint": "业务类型推断（如：两地牌过户费、车辆购置税、年检费）。无法判断时null",
  ...}"""
```
- `business_hint` 只要求推断"业务类型"，没有要求提取"该款项系付"内容

**2. 描述生成逻辑** (`backend/app/ai/tool_executor.py:410-437`)
```python
business_hint = receipt_data.get("business_hint", "")
description = receipt_data.get("description", "") or business_hint or ""

# 自动生成更可读的描述
auto_desc = self._build_payment_description(
    amount=amount, currency=currency,
    installment_name=term_label_hint or "",
    business_hint=business_hint,
    payee_name=payee_name if payment_type == "expense" else None,
    contract=contract,
)
description = description or auto_desc
```

**3. 描述拼接函数** (`backend/app/ai/tool_executor.py:139-161`)
```python
def _build_payment_description(self, amount, currency, installment_name="", business_hint="", ...):
    parts = [f"{symbol}{amt}"]
    if installment_name:
        parts.append(installment_name)
    # 补充业务上下文
    ctx = business_hint or ""
    if contract and contract.business_description:
        ctx = contract.business_description
    if ctx:
        parts.append(ctx[:20])
    return " ".join(parts)[:100]
```

### 问题总结
1. 收据分析时没有提取"该款项系付"字段
2. `business_hint` 只推断业务类型（两地牌相关费用），不提取具体用途
3. 最终 description 用了笼统的业务类型，而非收据上的具体描述

---

## 修复方案

### 方案：收据分析时提取"该款项系付"内容

#### 改动点 1：修改收据分析提示词

**文件**: `backend/app/ai/prompts.py`
**位置**: `RECEIPT_ANALYSIS_PROMPT`（约第259行）

新增字段 `payment_purpose`：
```python
RECEIPT_ANALYSIS_PROMPT = """...{
  "payment_purpose": "该款项系付的具体内容，原样照抄收据上的文字（如：深圳湾现牌定金、尾款等）。收据没有此字段则null",
  "business_hint": "业务类型推断（如：两地牌过户费、车辆购置税、年检费）。无法判断时null",
  ...}"""
```

#### 改动点 2：修改描述生成逻辑

**文件**: `backend/app/ai/tool_executor.py`
**位置**: `match_and_confirm_payment` 方法（约第410-437行）

优先使用 `payment_purpose`：
```python
payment_purpose = receipt_data.get("payment_purpose", "")
business_hint = receipt_data.get("business_hint", "")

# 优先级：payment_purpose > description > business_hint
description = receipt_data.get("description", "") or payment_purpose or business_hint or ""

# 自动生成更可读的描述
auto_desc = self._build_payment_description(
    amount=amount, currency=currency,
    installment_name=term_label_hint or "",
    business_hint=payment_purpose or business_hint,  # 优先用 payment_purpose
    payee_name=payee_name if payment_type == "expense" else None,
    contract=contract,
)
description = description or auto_desc
```

#### 改动点 3：同步修改 create_payment_record（如需要）

**文件**: `backend/app/ai/tool_executor.py`
**位置**: `create_payment_record` 方法（约第818行附近）

同样需要优先使用 `payment_purpose`。

### 修复后

- `description`: "¥50,000 深圳湾现牌定金"

## 风险评估

- **低风险**: 只是新增字段提取，不影响现有逻辑
- **向后兼容**: 如果收据没有"该款项系付"字段，会 fallback 到 business_hint
- **无需数据库变更**: 只修改提示词和代码逻辑
