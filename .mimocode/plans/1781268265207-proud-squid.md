# 修复：合同卡片按钮进入时 LLM 仍询问关联合同的问题

## 问题根因

`backend/app/ai/prompts_v2.py` 第 40 行 context_block 措辞太软：

```
你正在为此合同处理收支录入，所有操作默认关联此合同。如果用户的操作明显与当前合同无关（例如用户想为另一个合同录入），请向用户确认。
```

LLM 将"默认关联"理解为建议而非硬规则，凭证识别度低时按"主动追问"策略反问"关联到哪个合同"。prompt 没有告诉 LLM：合同 ID 和收支类型已被前端按钮锁定，是定向入口。

## 修改方案

**文件**: `backend/app/ai/prompts_v2.py` 第 40 行

**修改前**:
```
你正在为此合同处理收支录入，所有操作默认关联此合同。如果用户的操作明显与当前合同无关（例如用户想为另一个合同录入），请向用户确认。
```

**修改后**:
```
你正在为此合同处理收支录入。合同ID和收支类型已被用户从合同页面锁定，无需确认——所有操作必须关联此合同（使用上方的合同ID和合同编号），绝对不要询问用户"关联到哪个合同"或"是收入还是支出"。如果凭证识别度低（金额/币种/付款方/日期不清晰），要问的是这些业务字段，而不是合同归属。
```

关键改动：
1. 明确"定向入口"：合同 ID 和收支类型已被前端锁定
2. 硬约束："**必须**关联" + "**绝对不要询问**"
3. 明确禁止两类追问：合同归属、收支类型
4. 引导正确追问方向：凭证业务字段（金额/币种/付款方/日期）

## 验证方法

1. `cd backend && uv run uvicorn app.main:app --reload --host 0.0.0.0 --port 8000`
2. `cd frontend && npm run dev`
3. 从合同列表页点击某合同的"录入收入"按钮
4. 上传一张无法识别合同的普通银行转账截图，不输入文字
5. 预期：LLM 直接使用上下文中的合同进行录入，不再询问"关联到哪个合同"；如果凭证不清晰，问的是金额/币种/付款方/日期等业务字段
