# 收款账户管理功能（持久化版本）

## 需求分析
- 收款账户需要持久化存储（数据库）
- 支持扩展：银行账户、支付宝、微信等
- 管理员可增删，所有角色可查看
- 后续可能关联到收款记录

## 数据库设计

### 表结构：`payment_accounts`

```sql
CREATE TABLE IF NOT EXISTS payment_accounts (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(200) NOT NULL,           -- 账户名称（如：高山贸易有限公司-华侨银行）
    account_type    VARCHAR(20) NOT NULL,            -- 账户类型：bank/alipay/wechat/other
    bank_name       VARCHAR(100),                    -- 银行名称（bank类型必填）
    account_name    VARCHAR(200) NOT NULL,           -- 户名
    account_number  VARCHAR(100),                    -- 账号
    branch          VARCHAR(200),                    -- 网点
    address         VARCHAR(500),                    -- 地址
    phone           VARCHAR(50),                     -- 电话
    swift_code      VARCHAR(50),                     -- SWIFT Code
    fps_id          VARCHAR(50),                     -- 转数快FPS ID
    qr_code_url     VARCHAR(500),                    -- 收款二维码URL（支付宝/微信）
    is_default      BOOLEAN DEFAULT FALSE,           -- 是否默认收款账户
    sort_order      INTEGER DEFAULT 0,               -- 排序
    remarks         TEXT,                            -- 备注
    created_by      INTEGER REFERENCES users(id),
    created_at      TIMESTAMPTZ DEFAULT NOW(),
    updated_at      TIMESTAMPTZ DEFAULT NOW(),
    is_deleted      BOOLEAN DEFAULT FALSE,
    deleted_at      TIMESTAMPTZ
);

COMMENT ON TABLE payment_accounts IS '收款账户配置表';
COMMENT ON COLUMN payment_accounts.account_type IS '账户类型: bank(银行)/alipay(支付宝)/wechat(微信)/other(其他)';
```

### 索引
```sql
CREATE INDEX IF NOT EXISTS idx_payment_accounts_type ON payment_accounts(account_type) WHERE is_deleted = FALSE;
CREATE INDEX IF NOT EXISTS idx_payment_accounts_default ON payment_accounts(is_default) WHERE is_default = TRUE AND is_deleted = FALSE;
```

## 后端实现

### 1. Model：`backend/app/models/payment_account.py`
- 继承 `BaseModel`
- 字段对应表结构
- 关系：`created_by` -> User

### 2. Schema：`backend/app/schemas/payment_account.py`
- `PaymentAccountCreate`：创建请求
- `PaymentAccountUpdate`：更新请求
- `PaymentAccountResponse`：响应（不含软删字段）

### 3. Service：`backend/app/services/payment_account_service.py`
- `list_accounts(db)`：列出所有有效账户（按 sort_order 排序）
- `get_account(db, account_id)`：获取单个账户
- `create_account(db, data, user_id)`：创建账户
- `update_account(db, account_id, data)`：更新账户（预留，当前不开放编辑）
- `delete_account(db, account_id)`：软删账户

### 4. API：`backend/app/api/v1/payment_accounts.py`
- `GET /`：列出所有收款账户（所有角色）
- `POST /`：新增收款账户（仅 admin）
- `DELETE /{account_id}`：删除收款账户（仅 admin）
- 路由前缀：`/api/v1/payment-accounts`

### 5. 注册路由：`backend/app/main.py`
```python
from app.api.v1 import payment_accounts
app.include_router(payment_accounts.router, prefix=f"{settings.API_V1_STR}/payment-accounts", tags=["收款账户"])
```

### 6. 工具定义：`backend/app/ai/tool_executor.py`（可选）
- 新增 `list_payment_accounts` 工具，供 Agent 查询收款账户信息
- 新增 `get_payment_account` 工具，获取单个账户详情

## 前端实现

### 1. API 服务：`frontend/src/services/paymentAccount.ts`
- `paymentAccountApi.list()`：获取账户列表
- `paymentAccountApi.create(data)`：创建账户
- `paymentAccountApi.delete(id)`：删除账户

### 2. 页面：`frontend/src/pages/PaymentAccounts.tsx`
- 从 API 加载数据（非写死）
- 卡片式展示账户信息
- 管理员可添加/删除


### 4. 菜单：`frontend/src/components/Layout.tsx`

## 验证方案
1. 执行 SQL 创建表和初始数据
2. 启动后端：`cd backend && uv run uvicorn app.main:app --reload`
3. 启动前端：`cd frontend && npm run dev`
4. 测试：

   - 点击「添加账户」→ 弹窗填写信息 → 保存成功
