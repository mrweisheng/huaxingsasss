侧边栏 LOGO 区 + 小星助手 入口重构

问题诊断：当前 "华星资源" LOGO 与 "小星助手" 入口都用了 32×32 / 28×28 的金色渐变方块图标，色彩、形态、密度高度一致，眼睛找不到落点。 品牌锚（LOGO）和功能 CTA（助手入口）必须用完全不同的视觉语言区分主次。 下方给出 3 个方向，请选定后我再改 React 代码。

现状问题
LOGO 区：32×32 金色渐变方块 + 衬线"华"字 + 16px 标题
助手入口：28×28 金色渐变方块 + 星标 + 14px 标题 + 金色 Tag
→ 两者图标盒子、配色、版式同源，无视觉层级，用户进入侧边栏时无第一焦点。

个人推荐：方案 C「书脊 × 金牌浮岛」。 你这套系统的核心叙事是 "小星助手 = 智能录入主入口"（CLAUDE.md 里 Agent 模式被列为最高优先级）， C 把所有金色权重都押在 CTA，让用户进侧边栏第一眼就被它抓住；LOGO 用书脊形态退到二线作为身份印记。 视觉层级和业务优先级完全对齐，且实现成本最低、与现有深蓝品牌色无缝兼容。
