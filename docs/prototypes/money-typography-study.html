金额排版 · 5 种现代方案

现版 HK$140,000 飘的根因不是字号，而是 缺少层级、缺少底座、单一字体单一字重。本原型基于 Stripe · Mercury · Ramp · Wise · 银行收据等现代 UI 抽象出 5 类有效手法， 全部应用到同一份合同（胡少棟 · HK$140,000 / HK$250,000 · 56%）以便直接横评。

为什么现在的数字"飘"
单字体单字重
货币符号、整数、小数、单位全是 JetBrains Mono Bold，无对比
缺少底座
数字直接漂在白底卡片上，没有视觉锚点
死亡字号区间
20px 既不够小见精细，也不够大见冲击，最"温吞"的尺寸
无节奏对比
currency vs digits、integer vs decimal、当前 vs 总额 全平等

Stripe Dashboard 风 · 字号阶梯

Inter Tight · 货币上标 11px · 整数 26px · 小数 14px 灰

Stripe / Polar / Cal.com 都采用此手法：货币符号上标缩小、整数放大加粗、小数部分（哪怕是 .00）灰化缩小。 数字之间的字号阶梯本身就是层级。换 Inter Tight，因为它是为数字 UI 优化的几何无衬线， 20+px 区间的视觉密度优于 JetBrains Mono。整数颜色用 --brand-gold-deep #a87a18 比 #c9952b 在白底上更稳。

收据/银行流水风 · 底座 + 节奏

金色羊皮纸底 · 虚线分隔 · 左 LABEL 右 NUMBER 对齐

给数字一个"金融纸"底座—— 这是银行回单、ATM 凭条、Square 收据用了半个世纪的视觉语言，传达"这是钱、这是凭证"。 手法：fdf8ef 米黄底 + 左侧虚线齿条 + 三行对齐（已收/总额/未收 / 业务摘要）。 优点：底色本身就是视觉重量，数字不必再"喊"。还能把已收/总额/未收三个数同框对比。

重型数字盒 · Mercury / Ramp 风

深色金属底 · 金色数字 · 顶部金箔线 · 最强视觉冲击

Mercury Bank、Ramp、Brex 的核心账户余额都用"装数字进暗色容器"的手法：金额脱离白底， 坐在#0f1a2e 深蓝金属背景上，金色数字 + 顶部一道金箔细线（亮点高光）。 优点：视觉权重是 5 个方案里最强的，绝对不会"飘"。 代价：暗色块和卡片整体白调形成对比，风格更"科技/金融"，少一些"中式典雅"，与项目品牌色搭配良好但需评估接受度。

比例条锚定 · Wise / Linear 风

金额 + 进度同一个对象 · 分数式 已收/总额 · 进度条带末端高光锚点

Wise、Linear Insights、Notion 数据库进度条用的手法：把"金额"和"进度"合并成单一视觉对象， 数字直接坐在浅灰底框里，下方进度条带末端高光锚点（一个突出的小方块）， 暗示"钱走到这里了"。百分比做成 chip 徽章放右上角，与金额数字形成"主指标 / 辅助指标"的明确层级。 最适合 "已收/总额"是核心问题的场景 —— 信息密度高但不喧宾夺主。

方案 核心手法 视觉权重 信息密度 品牌契合 实现成本 适用判断
V0 现版 单字体单字重 ★★ ★★★ ★★★ — 对照基线
V1 Stripe 字号阶梯 + Inter Tight ★★★★ ★★★★ ★★★★ 低 · 仅 CSS 稳妥首选。改动小、风险低、效果立竿见影，跨币种数字对齐最稳
V2 收据风 金色底座 + 三行对齐 ★★★★★ ★★★★★ ★★★★★ 中 · 重构金额区 金融感最强，能同时展示已收/总额/未收。卡片高度比 V1 高 12~16px
V3 衬线奢华 Fraunces + Instrument Serif ★★★★ ★★★ ★★★★★ 低 · 加 2 个字体 气质独到，与「华星」品牌名气质最契合。需要接受"数字也用衬线"的设计取向

V5 比例条锚定 金额+进度合体 ★★★★ ★★★★★ ★★★★ 中 · 重构金额区 信息组织最佳，"已收/总额/进度/百分比"四合一。比 V1 高约 6px
我的推荐：
主选 V1 (Stripe 字号阶梯) —— 改动量最小、视觉提升立竿见影、跨币种稳定。
升级版选 V5 (比例条锚定) —— 如果你能接受合同卡片每个稍高 6~8px，V5 把金额、总额、进度、百分比融为一体，信息组织最高级。
追求品牌差异化选 V3 (Display Serif) —— 想要做一个"看一眼就知道是华星家系统"的差异化设计，这个最独特。
