# 小星助手 · UI 优化设计方案

> 基于主流 AI Agent 工具调研的三栏式布局重构方案
> 设计日期：2026-06-09 | 设计师：UI Designer

---

## 一、调研摘要

### 1.1 主流 AI Agent 工具布局趋势

| 工具 | 布局特征 | 可借鉴点 |
|------|---------|---------|
| **Cursor** | 三栏式：对话 + 代码 + 活动流 | 活动面板独立展示 Agent 行为，左右分栏 |
| **Claude** | 简洁对话 + 左侧历史 | 极简输入区设计，会话自动标题 |
| **ChatGPT** | 左侧历史 + 居中对话 | 清晰的历史列表管理 |
| **Perplexity** | 焦点模式切换 + 来源面板 | 顶部焦点选择器设计 |
| **Claude Code** | 活动流 + 工具调用追踪 | 独立的工具调用面板 |

### 1.2 核心设计模式（来自 2026 年 UX 研究）

1. **对话与活动分离**：聊天记录 ≠ Agent 活动日志，应有两个独立视图
2. **渐进式披露**：摘要 → 详情 → 原始数据，按需展开
3. **模式化交互**：类似 Perplexity 的焦点模式（通用/合同/收入/支出）
4. **数据引用透明**：Agent 引用的数据需实时可见，增强信任

## 二、当前页面问题分析

| # | 问题 | 严重程度 |
|---|------|---------|
| 1 | **欢迎页与聊天页体验割裂**：两种完全不同的布局（居中 vs 左对齐），切换时有跳变感 | 🔴 |
| 2 | **工具选择器重复出现**：欢迎页一套、底部一套，交互逻辑不一致 | 🔴 |
| 3 | **消息区域过窄**：max-width: 768px 在宽屏下浪费大量空间 | 🟡 |
| 4 | **AI 思考过程嵌入聊天气泡**：thoughts/tool_calls 挤在消息气泡中，信息密度过高 | 🔴 |
| 5 | **缺少数据引用面板**：用户看不到 AI 引用了哪些合同 / 付款记录 | 🔴 |
| 6 | **快捷建议只在欢迎页**：聊天过程中无法看到快捷建议 | 🟡 |
| 7 | **无会话搜索**：会话数量增多后难以找到历史内容 | 🟡 |
| 8 | **三种工具被固定为"先选再用"模式**：增加用户操作摩擦 | 🟡 |

## 三、新布局方案：三栏式结构

```
┌─────────────────────────────────────────────────────────────┐
│  ┌──────────┐  ┌───────────────────────┐  ┌────────────┐   │
│  │ 左侧面板  │  │     中央主区域        │  │ 右侧面板    │   │
│  │ (200px)  │  │    (flex: 1)          │  │ (220px)    │   │
│  │          │  │                       │  │            │   │
│  │ ★ 小星AI │  │  ┌─────────────────┐  │  │ 活动面板   │   │
│  │ [+新对话]│  │  │  欢迎页/消息列表  │  │  │            │   │
│  │ 搜索...  │  │  │                 │  │  │ ⚡思考链    │   │
│  │          │  │  │  ★ 你好我是小星  │  │  │            │   │
│  │ 会话列表 │  │  │  能力卡片...     │  │  │ 📊数据引用 │   │
│  │          │  │  │  输入区域...     │  │  │            │   │
│  │ ▸ 合同查  │  │  └─────────────────┘  │  │ 🔧工具调用 │   │
│  │ ▸ 收入录  │  │                       │  │            │   │
│  │ ▸ 支出管  │  │  ┌─────────────────┐  │  │ 历史记录   │   │
│  │          │  │  │  底部输入区       │  │  │            │   │
│  └──────────┘  │  │  [模式] 📎 输入  │  │  └────────────┘   │
│                │  │  [发送→]         │  │                    │
│                │  └─────────────────┘  │                    │
│                └───────────────────────┘                    │
└─────────────────────────────────────────────────────────────┘
```

### 3.1 左侧面板 — 会话管理（200px）

**设计参考**：ChatGPT / Claude 的侧边栏
- 深色背景 (#0f1a2e) 延续品牌侧边栏风格
- 顶部：小星 Logo + "小星助手" + AI 标签
- "+新对话"按钮：一键创建新会话
- 会话搜索框：支持按标题/内容搜索历史
- 会话列表：自动生成标题 + 日期，最后使用的置顶

### 3.2 中央主区域 — 对话核心（flex: 1）

**设计参考**：Claude / Perplexity

#### 欢迎态（空会话）
- 居中布局：金色星星 Logo + "你好，我是小星" + 副标题
- **能力卡片**：4 张轻量级语义色卡片（合同、收入、支出、问答）代替纯文字标语
- **模式选择器**：通用问答 / 录合同 / 录收入 / 录支出，类似 Perplexity 焦点模式
- 居中输入框：聚焦时品牌色边框光晕
- 快捷建议：3 个可点击建议胶囊

#### 聊天态（有消息）
- 保持与欢迎态相同的布局骨架
- 欢迎内容淡出，消息列表显示
- 消息气泡：用户右对齐（深蓝渐变）、AI 左对齐（白底）
- 思考链/tool_calls**不再嵌入消息气泡**，转移到右侧活动面板
- 快捷建议依旧显示在消息流尾部
- 底部输入区始终固定

### 3.3 右侧面板 — 活动流（220px，可收起）

**设计参考**：Cursor 活动面板 / Claude Code 活动流

这是本次重构**最核心的新增组件**，包含三个信息层：

#### ① 思考链（Thinking Chain）
- 实时展示 AI 正在处理的步骤（类似 Claude Code 的步骤指示器）
- 已完成步骤：绿色圆点 ✓
- 进行中步骤：金色脉冲动画 ◌
- 每个步骤显示简短描述

#### ② 数据引用（Data Reference）
- 展示 AI 从系统检索到的数据摘要
- 如：匹配合同 3 份、待付款 ¥124,800、已逾期 2 笔
- 数字可点击跳转到对应业务页面
- 用户一眼了解 AI 的回答基于什么数据

#### ③ 工具调用日志（Tool Call Log）
- 记录每个工具调用的状态
- 显示：工具名称、耗时、状态（成功/处理中/失败）
- 可折叠查看详细参数和返回结果
- 支持点击"重试"或"查看详情"

## 四、交互流程变更

### 4.1 模式选择器（替代旧 ToolSelector）

```
旧方案：工具选择器是独立组件，在欢迎页和底部重复出现
       用户必须先"选一个工具"，再提问，增加操作步骤

新方案：模式选择器融入输入区上方，类似 Perplexity 的 Focus 模式
       - 默认"通用问答"模式
       - 选择"录合同"时：输入框 placeholder 变为"上传合同文件或粘贴合同内容..."
       - 选择"录收入"时：提示"请上传收入凭证图片或填写金额"
       - 模式以标签形式显示在输入框左侧，可随时切换或清除
       - 不再需要独立的 ToolSelector 组件
```

### 4.2 活动面板交互

```
- 默认展开
- 点击右上角 ◀ 按钮可收起为窄条（只保留图标）
- 思考链自动滚动，最新步骤始终可见
- 数据引用块在每次 AI 检索数据后更新
- 工具调用显示执行时间，异常调用可点击"重试"
```

### 4.3 统一布局骨架

```
旧方案：两套独立布局
  - 欢迎页：居中 flex 布局
  - 聊天页：flex:1 + max-width 768px
  - 切换时有视觉跳变

新方案：一套布局骨架贯穿始终
  - 左侧面板始终存在
  - 右侧面板始终存在
  - 中央区域：欢迎态 → 聊天态 → 过渡动画
  - 底部输入区始终存在（欢迎态时显示在中间，聊天时固定在底部）
```

## 五、设计 Token 适配

新方案完全复用现有品牌设计系统，无需新增色值：

```css
/* 已存在的 Token 即可覆盖 */
--brand-deep: #0f1a2e;       /* 左侧面板背景 */
--brand-primary: #1e3a5f;    /* 用户气泡、发送按钮 */
--brand-gold: #c9952b;       /* AI 头像、模式选中态 */
--color-success: #0d9488;    /* 完成状态圆点 */
--color-danger: #dc2626;     /* 逾期数据高亮 */
--color-info: #2563eb;       /* 数据引用图标 */
```

## 六、实施建议（按优先级排序）
